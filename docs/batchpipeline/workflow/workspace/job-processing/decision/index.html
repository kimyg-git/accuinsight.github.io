decision

#Definition

decision 노드에 입력하는 조건에 따라 N개의 Node로 작업 flow를 분기시킨다. 선행 노드의 결과에 따라 실행될 후행 노드가 다를 경우 선택한다.
좌측 [Flow구성]노드 중 [decision]노드를 drag & drop 한 후 Property 항목을 입력한다.

#Set

[setting], [scheduler], [parameter] 설정은 [워크플로우 생성] > [설정]을 참고한다.

#property

[Node Description] 작성 중인 노드명 입력

  1. switch : 조건문과 이동할 action을 입력한다.
    • case : 조건 입력
    • to : 조건에 따라 수행할 후속 노드 선택(콤보박스)
  2. default : 기본으로 이동할 action
    • to : 노드 선택

#Example

HDFS에 특정 파일 저장여부를 체크하여 파일이 있을 경우 "fileexist_yes", 없을 경우 "no_files"라는 0byte 파일을 생성한다. ETL 수행결과가 정상적으로 생성되었는지 확인하는 검증로직 등으로 응용할 수 있다.

  1. kbtest 클러스터에 검사대상 파일을 적재한다(/tmp/test-ods/AIR_HOUR_2019_2.csv).
  2. [decision] 및 [fs] 노드를 생성한다.
  3. [decision] 노드를 아래와 같이 설정한다.
  • 경로에 파일이 있을 경우 [fs_fileexist]로 이동
    • case : ${fs:exists(("/tmp/test-ods/AIR_HOUR_2019_2.csv")) == "true"}
    • to : fs_fileexist
  • 경로에 파일이 없을 경우 [fs_nofiles]로 이동
    • case : ${fs:exists(("/tmp/test-ods/AIR_HOUR_2019_2.csv")) == "false"}
    • to : fs_nofiles

[Note] : oozie el function
case 입력 구문은 oozie el function 으로 세부 사항은 아래를 참고한다.
https://oozie.apache.org/docs/4.3.0/WorkflowFunctionalSpec.html#a4.2.2_Basic_EL_Functions

  1. [fs]노드(fs_fileexist, fs_nofiles)를 아래와 같이 설정한다.
  • fs_fileexist : property "5.touchz" > HDFS File Browser에서 "hdfs://kbtest-accu-hdfs-nn.suka:9000/tmp/test-ods"경로 설정 후 "/fileexist.yes" 입력
  • fs_nofiles : property "5.touchz" > HDFS File Browser에서 "hdfs://kbtest-accu-hdfs-nn.suka:9000/tmp/test-ods"경로 설정 후 "/no_files" 입력
  1. 실행결과는 다음과 같다.

  • 데이터가 있을 경우 지정한 경로에 fileexist.yes 생성

  • 데이터가 없을 경우 지정한 경로에 no_files 생성
