fs

#Definition

HDFS상의 작업을 수행한다. delete, mkdir, move, touchz, chmod,chgrp 등 hadoop fs syntax로 수행할 수 있는 작업이 가능하다. 좌측 [Flow구성]노드 중 [fs]노드를 drag & drop 한 후 Property 항목을 입력한다. Property 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 Property 항목을 볼 수 있다.

#Set

[setting], [scheduler], [parameter] 설정은 [워크플로우 생성] > [설정]을 참고한다.

#property

[Node Description] 작성 중인 노드명 입력

  1. retry : 재수행 횟수 및 간격을 실행한다.
    • max : 재수행 횟 수
    • period : 재수행 간격
  2. delete : 삭제할 hdfs path 설정
    • skipTrash: 완전 삭제(hadoop에 설정된 휴지통을 거치지 않음)
  3. mkdir : 생성할 hdfs directory path 설정
  4. move : source path, target path 설정으로 파일 이동
  5. touchz : 파일크기가 0인 파일을 생성한다.
  6. chmod : path: 권한 설정 경로 : permissions: 권한유형 : recursive: 재귀여부
  7. chgrp : path: 그룹 설정 경로 : recursive: 재귀여부 : group: 그룹명
  8. forceOK : 실패 시 강제 OK 처리 여부

#Example

  1. "2.delete" property 설정 후 워크플로우 실행시 선택한 폴더가 삭제된다.

  1. "3.mkdir" property 설정 후 워크플로우 실행시 선택한 폴더가 삭제된다.
