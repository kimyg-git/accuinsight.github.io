#배포

#학습된 모델 배포

  • 모델 학습이 완료되면 학습된 모델을 API서버로 배포할 수 있다.
  • API 서버는 이미지 파일을 입력으로 받으면, 해당 이미지의 분류 결과를 정수로 반환한다.
  • 반환하는 정수 값은 label csv 파일에 입력했던 값 중 하나이다.

  • 배포를 위해, 학습 상세 팝업 우측 상단에 있는 배포 버튼을 클릭한다.

  • 학습 관리 페이지에서 로켓 버튼을 눌러도 동일하게 동작한다. 마찬가지로, 로켓 버튼은 완료된 학습에 대해서만 표시된다.

  • 다음은 배포 서버를 생성하는 과정이다.

a. 상단의 배포 명과 배포 설명을 입력한다.

- 배포 서버 명: deploy-cpu
- 배포 서버 설명: model deploy demo

b. 배포에 활용할 인스턴스를 선택한다. gpu개수 제한으로 여기에서는 cpu를 선택한다.

c. Pb파일을 선택한다. Pb파일은 학습의 결과로 생성된 모델 파일로, 오른쪽의 폴더 아이콘을 누르면 browse할 수 있다.

바로 모델을 확인할 수 있다. saved_model.pb 파일을 선택한 후 확인을 클릭한다.

d. 모든 내용을 입력한 후 생성 버튼을 누르면 배포 서버가 생성된다.

  • 배포 성공 팝업에서 예 버튼을 누르면 배포 관리 화면으로 이동한다. 아니오를 누르면 학습 페이지에 남게 된다.
  • 화면 우측 상단의 분석 메뉴를 통해, 학습 페이지에서 배포 관리 페이지로 이동할 수 있다.

#샘플 예측

  • 배포된 서버는 배포 관리 화면에서 확인 가능하다.
  • 배포 서버 명을 누르면 샘플 예측을 위한 예측 관리 페이지로 이동된다.

  • 이 페이지에서 해당 배포 서버에 대해 간단한 테스트를 해볼 수 있다.
  • 예측 생성 버튼을 눌러 예측 생성 팝업으로 진입한다.

  • 다음은 예측을 생성하는 과정이다.

a. 예측 작업 명과 예측 작업 설명을 입력한다.

- 예측 작업 명: pred-demo 
    - 예측 작업 설명: prediction demo

b. 테스트 데이터 선택을 위해 폴더 아이콘을 누른다.

테스트 데이터로 테스트용 이미지 데이터를 선택한다.

폴더 경로

mnt > project > image-classification324_edu01 (사용자에 따라 다름) > data > pred-image > pred-image(또는 processed-pred-image{날짜})

폴더를 선택하면 해당 폴더에 있는 모든 이미지로 예측 해볼 수 있다.

c. 데이터 선택 후 예시 리프레시 버튼을 누르면 해당 폴더의 이미지가 랜덤으로 하나씩 보여진다.

d. 샘플 예측 버튼을 누르면 샘플 이미지를 배포 서버가 어떻게 예측했는지 볼 수 있다. label을 정수로 넣었기 때문에, 정수로 결과를 반환한다.

e. 생성 버튼을 클릭하면 선택한 테스트 데이터로 예측을 진행한다.

  • 예측이 생성되었다. 자세한 예측 결과는 예측 작업 명을 누르면 볼 수 있다.
