소스 저장소

#소스 저장소 관리

  • 프로젝트는 논리 그룹으로 구성된 워크스페이스, 저장소, 배포 리소스를 그룹화하여 관리합니다.
  • 소스 저장소는 분석 작업중인 소스코드 형상관리를 위한 저장소이며, 워크스페이스 리소스가 삭제되더라도 소스 저장소에 저장된 소스코드는 유지됩니다.

소스 저장소 목록

  1. 필터 조회 영역
    • 검색필터 버튼을 클릭하면 입력창이 활성화 됩니다. 등록일과 기간을 가지고 검색
  2. Gitlab
    • Gitlab으로 연결
  3. 워크스페이스로 다운로드
    • 저장된 소스의 마지막 버전을 워크스페이스로 다운로드
  4. 삭제
    • 해당 소스를 삭제

#소스 저장소 > 상세정보 관리

소스 저장소 상세페이지는 해당 소스의 상세 정보를 조회하고, 업데이트 히스토리를 관리할 수 있습니다.

  1. 삭제/새로고침
  2. 요약
    • 소스 저장소 저장 시 설정된 정보와 저장한 파일 목록을 조회
  3. 히스토리
    • 소스 저장소에 저장된 업데이트 히스토리 목록을 조회할 수 있으며, 특정 리비전의 소스를 다운로드 또는 워크스페이스로 다운로드 할 수 있음
